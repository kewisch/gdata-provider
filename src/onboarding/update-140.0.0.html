Hi beta users! This is the page that release channel users will see when they upgrade to the new version. It contains a migration section where calendars can be upgraded to the new format. Please give it a whirl and let me know how it goes.

You will not see the section if you don't have legacy calendars in your list. If you want to try it out nevertheless you can keep this tab open, subscribe to a legacy calendar and then reload this page. Alternatively test the upgrade in the calendar properties of a legacy calendar.

Further please make sure the legacy calendars still work as expected when subscribing to them.

Welcome to version 140.0.0 of the Provider for Google Calendar. A while back I asked for beta testers, and now the changes we tested together are making their way into the release version.

This update introduces the new approach while still keeping the old one available. Both are bundled into this release so you can try the new features with confidence, and if something does not work as expected you can easily switch back.

⚠️ Upgrade your calendars

To use the new features, you will need to update your calendars. For now this step is optional, but I strongly recommend upgrading now so that any issues can be identified and fixed early. The old approach will be removed in a future release.

If you want to go back, unsubscribe from the calendar and re-subscribe using the legacy option.

What's new?

Here is a selection of features and fixes in the new approach. The code has been rebuilt from the ground up using a new set of calendar APIs, which I plan to submit to Thunderbird core so that all add-on developers can benefit from them.

  • Attachments are now linked on events (read-only)
  • You can add or remove video conference links directly from your events
  • Events you are invited to that are marked as public will instead be created as private in your calendar, to protect your privacy.
  • Birthday events are now supported, just subscribe to the birthday calendar to see them.
  • A whole class of issues related to modifying events you are invited to is resolved, as the Provider will just update the properties you change, not attempt to replace the whole event.
  • Dark mode looks better across dialogs, options, and setup screens.
  • Making many quick changes, like deleting multiple events, will no longer cause rate limiting errors.
  • Event invitations and attendees are handled more reliably.
  • Invitations that use Windows timezone names are now supported more consistently.
